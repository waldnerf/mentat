# Mentat Knowledge Lifecycle v1.0

## Purpose

The Mentat Knowledge Lifecycle defines how ideas evolve from initial thoughts into structured knowledge and reusable outputs.

The lifecycle ensures that knowledge is developed intentionally.

Not every idea follows the same path.

Some become reusable knowledge assets.

Some become publications.

The decision is deliberate.

# 1. Capture

## Objective

Preserve ideas before they disappear.

Capture should be lightweight.

Examples:

- observations
- questions
- research notes
- conversations
- hypotheses
- insights

At this stage, optimise for preservation.

Do not optimise for perfection.

Location:


knowledge/inbox/


---

# 2. Understand

## Objective

Clarify the meaning and intent behind an idea.

Identify:

- why it matters
- who it helps
- what problem it addresses
- what decision it supports
- what context surrounds it

Questions:

- What is the insight?
- Why is it valuable?
- What could this become?

---

# 3. Structure

## Objective

Transform an idea into an organised thought.

Create a Markdown scaffold.

Possible structures:

- framework
- model
- concept
- principle
- article outline
- decision record

Location:


knowledge/explorations/


---

# 4. Connect

## Objective

Place the idea within the existing knowledge system.

Evaluate:

- related concepts
- existing assets
- dependencies
- contradictions
- opportunities for extension

Knowledge gains value through connection.

---

# 5. Decide

## Objective

Determine the appropriate destination.

Every idea reaches a decision point.

Possible outcomes:

## Knowledge Asset

Use when the idea:

- is reusable
- explains a broader pattern
- supports future thinking
- can generate multiple artefacts

Move to:


knowledge/assets/


---

## Publication

Use when the idea:

- expresses a timely perspective
- communicates an observation
- has value as a standalone message
- does not require long-term maintenance

Move to:


publications/


---

# 6. Create

## Objective

Develop the canonical Markdown asset.

A knowledge asset should contain:

- purpose
- context
- core idea
- structure
- application
- examples
- relationships
- evolution notes

The Markdown file becomes the source of truth.

---

# 7. Generate

## Objective

Transform knowledge into artefacts.

Examples:

Knowledge asset:


knowledge/assets/frameworks/ai-operating-model.md


Generates:


artefacts/

website/
presentation/
diagrams/
social/
documentation/


Artefacts are expressions of knowledge.

They are not the source.

---

# 8. Publish

## Objective

Share knowledge through appropriate channels.

Examples:

- website
- LinkedIn
- presentations
- talks
- documentation

Publishing is distribution.

It does not replace the canonical source.

---

# 9. Reuse

## Objective

Increase the value of knowledge through application.

Reuse includes:

- extending
- combining
- teaching
- implementing
- referencing

The strongest assets create multiple downstream outcomes.

---

# 10. Improve

## Objective

Continuously evolve the system.

Improve:

- clarity
- structure
- accuracy
- connections
- reuse

Update canonical knowledge.

Regenerate artefacts.

# Lifecycle Principle

Knowledge should evolve deliberately.
Do not force every idea into a framework.
Do not create directly in publishing formats.
Start with thinking.
End with reusable value.
