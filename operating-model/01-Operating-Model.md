# Mentat Operating Model v1.0

## Purpose

Mentat is a personal operating model for thinking, building and sharing reusable knowledge.

It defines how ideas are captured, structured, connected, transformed and reused.

Mentat is a living knowledge system.

Its purpose is to increase the quality, accessibility and reuse of knowledge over time.

---

# Core Philosophy

## Knowledge Is a System

Knowledge should not exist as isolated documents.

The value of knowledge increases through:

- structure
- connections
- reuse
- refinement
- application

Every asset should strengthen the wider system.

---

# Knowledge Is Markdown First

Markdown is the primary environment for thinking and knowledge development.

Markdown provides:

- transparency
- portability
- version control
- human readability
- AI compatibility

The Markdown file is the working source.

Artefacts are generated from it.

---

# Core Principles

## 1. Intent Before Action

Understanding precedes execution.

Before creating anything:

- clarify intent
- understand the objective
- identify the audience
- identify constraints
- confirm assumptions

Thinking precedes production.

---

## 2. Structure Before Expression

Ideas become valuable when structured.

Prefer:

- frameworks
- models
- taxonomies
- decision trees
- architectures

Structure creates reusable understanding.

---

## 3. Connect Before Create

New knowledge should connect to existing knowledge.

Before creating a new asset:

Ask:

- Does this already exist?
- Can an existing asset be improved?
- Can this extend an existing concept?
- Where does this belong?

Connections create compounding value.

---

## 4. Stewardship Before Creation

Knowledge requires maintenance.

Responsibilities include:

- improving
- consolidating
- simplifying
- connecting
- removing duplication

Creation should follow understanding and connection.

---

## 5. Reuse Before Reinvention

Knowledge should be designed for reuse.

Prefer:

- templates
- patterns
- frameworks
- components
- libraries

Avoid isolated outputs.

---

## 6. Signal Over Noise

Mentat exists to increase signal.

Every asset should:

- improve understanding
- reduce complexity
- enable better decisions

---

## 7. Systems Over Features

Optimise the ecosystem, not the individual output.

Consider:

- future reuse
- maintainability
- consistency
- scalability

---

## 8. Not Every Idea Becomes a Knowledge Asset

Ideas have different destinations.

Some ideas mature into:

- principles
- frameworks
- models
- concepts

Others become:

- LinkedIn posts
- articles
- talks
- reflections

The decision is intentional.

Every published idea should originate from deliberate thinking.

# Canonical Knowledge Principle

Every important concept should have one canonical Markdown source.

The canonical asset contains the durable understanding.

Derived artefacts are generated from that source.

Rule:

> Update the knowledge. Regenerate the artefacts.

---

# Success Definition

A successful Mentat asset:

- creates understanding
- connects to existing knowledge
- can be reused
- can generate multiple artefacts
- improves the overall system
